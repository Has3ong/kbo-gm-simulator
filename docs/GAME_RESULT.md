# 경기 결과 처리 (Game Result Processing)

> 경기 시뮬레이션이 완료된 직후 순차적으로 실행되는 후처리 파이프라인.

---

## 전체 흐름

```
경기 시뮬레이션 완료
        ↓
  경기 결과 저장
        ↓
  선수/팀 경기 기록 저장
        ↓
  선수/팀 시즌 기록 누적
        ↓
     순위 갱신
        ↓
 피로도/부상/컨디션 반영
        ↓
  재정/팬 반응 반영
        ↓
  뉴스/이벤트 생성
        ↓
 경기 처리 완료 상태 저장
```

---

## 단계별 상세

### Step 1 — 경기 결과 저장
| 항목 | 내용 |
|------|------|
| 상태 | ✅ 구현 완료 |
| 대상 테이블 | `GAME` |
| 처리 내용 | `HOME_SCORE`, `AWAY_SCORE`, `WIN_TM_ID`, `GAME_STTS_CD = '03'(완료)` 업데이트 |
| 점수 산출 | 팀 CNT/PWR 평균 vs 상대 투구력(VEL/CTL) → 포아송(Poisson) 분포로 득점 생성 |

---

### Step 2 — 선수/팀 경기 기록 저장
| 항목 | 내용 |
|------|------|
| 상태 | ✅ 구현 완료 |
| 대상 테이블 | `PLR_BATR_GAME_REC`, `PLR_PTCH_GAME_REC` |
| 처리 내용 | 경기에 출전한 모든 선수의 타석·투구 기록 INSERT IGNORE |

#### 타자 기록 생성 공식

| 항목 | 공식 |
|------|------|
| 타석 (PA) | 4 or 5 (랜덤) |
| 볼넷 (BB) | 확률 8% |
| 안타 (H) | 이항 분포, 히트확률 = 0.150 + (CNT-20)/60 × 0.200 (범위 0.100~0.400) |
| 홈런 (HR) | 이항 분포, 홈런확률 = 0.005 + (PWR-20)/60 × 0.045 (범위 0.001~0.100) |
| 2루타 (DOBL) | 비홈런 안타 × 20% |
| 3루타 (TRPL) | 비2루타 안타 × 3% |
| 도루 (SB) | 확률 = max(0, (STL-20)/60 × 0.15) |

#### 투수 역할 체계 (TM_ROTATION / TM_BULLPEN 기반)

| ROLE_CD | 명칭 | 이닝 할당 | 특이사항 |
|---------|------|----------|---------|
| SP | 선발 | STM 기반 4~8이닝 | TM_ROTATION에서 무작위 선택 |
| MR | 중간계투 | SP 이후 남은 이닝 분배 | 팀 승패 무관 등판 |
| SU | 셋업맨 | CL 직전 3~5아웃 | 세이브 상황(승리+점수차≤3)에만 등판 |
| CL | 마무리 | 마지막 3아웃 | 세이브 상황에만 등판, 세이브 기록 |

**SP 이닝 계산**
```
기본 아웃 = 12 + round((STM - 20) / 60 × 9)
실제 아웃 = 기본 아웃 + 랜덤(−2 ~ +2) → 클램프 [12, 24]

STM=20 → 12아웃(4이닝) / STM=50 → 17아웃(5.2이닝) / STM=80 → 21아웃(7이닝)
```

**W/L/SV/HLD 판정**

| 판정 | 조건 |
|------|------|
| SP 승 | 팀 승리 + SP 15아웃(5이닝) 이상 |
| 계투 승 | 팀 승리 + SP 5이닝 미만 → 첫 번째 비-CL 계투 |
| SP 패 | 팀 패배 (단순화) |
| SV (세이브) | 팀 승리 + 점수차 ≤ 3 + 마지막 투수(CL) |
| HLD (홀드) | 세이브 상황 + SU 등판 + CL이 SV 기록 |

**특수 기록 (SP 전용)**

| 기록 | 조건 |
|------|------|
| QS | IP ≥ 18아웃(6이닝) + ER ≤ 3 |
| CG | SP 단독으로 27아웃 처리 |
| SHO | CG + 총 실점 = 0 |
| NH | CG + 피안타 = 0 |
| PG | NH + BB = 0 |

---

### Step 3 — 선수/팀 시즌 기록 누적
| 항목 | 내용 |
|------|------|
| 상태 | ✅ 구현 완료 |
| 대상 테이블 | `PLR_BATR_SSNT_REC`, `PLR_PTCH_SSNT_REC` |
| 처리 내용 | 경기 기록을 시즌 누적 집계에 ON DUPLICATE KEY UPDATE로 반영 |
| ERA/WHIP | 경기 직후 시즌 누계 기준으로 재계산 (`ER×27/IP_OUT`, `(BB+H)×3/IP_OUT`) |

---

### Step 4 — 순위 갱신
| 항목 | 내용 |
|------|------|
| 상태 | ✅ 구현 완료 |
| 대상 테이블 | `STND` |
| 처리 내용 | W/L/T/RS/RA/RUN_DIFF/PCT 갱신 → 승률 내림차순으로 STND_RNK/GB 재계산 |

---

### Step 5 — 피로도/부상/컨디션 반영
| 항목 | 내용 |
|------|------|
| 상태 | 구현 예정 |
| 대상 테이블 | `PLR_FATG_COND`, `PLR` |

#### 경기 출전 선수 처리

| 포지션 | 피로도 증가 | 컨디션 변동 |
|--------|------------|------------|
| 선발투수 (이닝 수 기반) | 투구 이닝 × 8 + 투구 수 × 0.3 | -5 ~ -15 랜덤 |
| 불펜투수 | 투구 이닝 × 10 | -3 ~ -10 랜덤 |
| 타자/수비 | 5 + 게임 연장 여부 × 2 | -2 ~ +5 랜덤 |

- `STM(체력)` 능력치가 높을수록 피로도 증가량 감소 (STM 60 이상: 0.8배, STM 40 이하: 1.2배)
- 히든 능력치 `IRK(부상위험도)`가 높으면 부상 확률 증가
- 피로도 90 이상 + 부상위험도 높음: 부상 판정 확률 상승

#### 경기 미출전 선수 처리

| 구분 | 피로도 | 컨디션 |
|------|--------|--------|
| 1군 벤치 | -3 감소 | +3 ~ +8 랜덤 증가 |
| 2군 선수 | -5 감소 | +5 ~ +10 랜덤 증가 |

- 쉬는 선수는 피로도가 감소하고 컨디션이 회복될 확률이 높음
- 특성 `GRND(악바리)`: 컨디션 낮아도 능력치 감소 폭 작음

#### 부상 판정 로직

```
부상 확률 = 기본확률(0.5%) × (1 + (피로도 - 70) / 100) × IRK_계수
IRK_계수: IRK 1~5=0.7, 6~10=1.0, 11~15=1.3, 16~20=1.7
```

### Step 6 — 재정/팬 반응 반영
| 항목 | 내용 |
|------|------|
| 상태 | 구현 예정 |
| 대상 테이블 | `TM_FNC_SSNT`, `STND` |
| 처리 내용 | 승리 시 방송국 승리 수당 적립 + 관중 수입 계산 + 팬 만족도 갱신 |
| 규칙 | 주말 경기는 관중 × 1.3 배율 적용, 원정 승리는 팬 만족도 +1.5 |

---

### Step 7 — 뉴스/이벤트 생성
| 항목 | 내용 |
|------|------|
| 상태 | 구현 예정 |
| 대상 테이블 | `SSNT_EVNT` |
| 생성 조건 | 부상 발생, 사이클히트, 완봉, 역전승, 기록 달성 등 주요 사건 |
| 이벤트 유형 | `INJ(부상)`, `REC(기록)`, `FAN(팬반응)`, `NEWS(일반)` |
| 초기 상태 | `RD_YN = '0'` (미읽음) |

---

### Step 8 — 경기 처리 완료 상태 저장
| 항목 | 내용 |
|------|------|
| 상태 | 구현 예정 |
| 대상 테이블 | `GAME` |
| 처리 내용 | `GAME_STTS_CD`를 `'03'(완료)`으로 확정, 이후 `진행하기` 버튼 활성화 |

## 능력치 보정 공식 (피로도/컨디션 → 실제 능력치)

```
실제능력치 = 기본능력치 × 피로보정 × 컨디보정

피로보정:
  피로도  0~ 50: 1.00 (영향 없음)
  피로도 51~ 70: 1.00 ~ 0.95 (선형 감소)
  피로도 71~ 90: 0.95 ~ 0.85 (선형 감소)
  피로도 91~100: 0.85 ~ 0.75 (급격 감소)

컨디보정:
  컨디션 71~100: 1.00 ~ 1.05 (소폭 상승)
  컨디션 41~ 70: 1.00 ~ 0.95 (선형 감소)
  컨디션  1~ 40: 0.95 ~ 0.80 (급격 감소)

특성 GRND(악바리): 컨디션 하락 보정 계수 0.5배 (피해 절감)
```
