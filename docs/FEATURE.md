# KBO 단장 시뮬레이터 — 기능 목록

유저(단장)가 직접 조작하거나 화면에서 확인하는 게임 기능을 정리한다.  
내부 시뮬레이션 공식·수치 기준은 `Rule.md` 참조.

---

## 1. 시즌 진행

### 1.1 시즌 상태

| 상태 코드 | 상태명 | 주요 가능 행동 |
|-----------|--------|----------------|
| PRE | 프리시즌 | 방송국 계약, 감독·코치 선임, 외국인 선수 계약, 스프링 캠프, 로스터 확정 |
| REG | 정규시즌 | 경기 진행, 트레이드, 콜업·말소 |
| POST | 포스트시즌 | 엔트리 고정, 라운드 진행 |
| CMPL | 완료 | 기록 확정, 수상 확인 |
| OFF | 오프시즌 | FA 계약, 방출, 드래프트, 연봉 협상 |

### 1.2 프리시즌(PRE) 필수 이벤트 순서

시즌 시작 직후 CUR_DT = 2월 1일. 아래 순서로 필수 이벤트를 완료해야 정규시즌이 시작된다.

| 순서 | 이벤트 | 완료 조건 | GAME_CFG 키 |
|------|--------|-----------|-------------|
| 1 | 방송국 스폰서 선택 | SBS/KBS/MBC 중 1개 선택 | `BRDCST_SPNSR` |
| 2 | 감독·코치 선임 | 감독 1명 + 코치 최대 2명 계약 | `STFF_HIRED` = 1 |
| 3 | 외국인 선수 계약 | 2/1~2/10 기간 중 진행 (선택) | — |
| 4 | 스프링 캠프 선택 | 7개 장소 중 1개 선택 → CUR_DT 3/15로 점프 | `SPRING_CAMP_DONE` = 1 |
| 5 | 로스터 확정 | 1군 엔트리 확정, 타순·선발로테이션 설정 | `ROSTER_CONFIRMED` = 1 |

모든 필수 이벤트 완료 후 CUR_DT가 REG_SSNT_BGNG_DT(3/22 등) 도달 시 자동으로 REG 상태로 전환된다.

### 1.3 진행 버튼

| 버튼 | 동작 | 표시 조건 |
|------|------|-----------|
| 다음주까지 진행하기 | 다음 주 월요일까지 하루씩 진행 (SSE 스트리밍) | PRE·REG·POST 공통 |
| 경기 시작하기 | 오늘 예정 경기 시뮬레이션 | 오늘 미완료 경기 있을 때 |

각 단계에서 부상·이벤트·FA 마감일이 날짜 단위로 처리된다.

## 2. 경기 결과 확인

- 경기는 **자동 시뮬레이션**으로 진행 (타석 단위 조작 없음)
- 유저는 경기 결과 요약(점수, 승패, 주요 선수 기록)만 확인
- 경기 후 선수 기록(안타·홈런·이닝 등) 갱신 확인 가능

## 3. 로스터 관리

### 3.1 엔트리 규정

| 구분 | 규모 |
|------|------|
| 1군 등록 엔트리 | 최대 29인 / 최소 20인 |
| 외국인 선수 | 팀당 최대 3명 |
| 포스트시즌 엔트리 | 정규시즌 마지막 1군 엔트리로 고정 |

### 3.2 로스터 확정 (프리시즌 필수)

- 3월 15일(스프링 캠프 종료 시점) 로스터 확정 이벤트(`EVNT_TYPE_CD = RCNF`) 발생
- 유저는 로스터 확정 화면(`/roster-confirm`)으로 이동 — **내비게이션 바 없는 풀스크린 레이아웃**

#### 화면 구성

```
[AppBar] 시즌으로 돌아가기 | 로스터 확정 | 1군 N명 칩 | 외국인 N/3 칩 | 확정하기 버튼
┌───────────────────────────────────┬────────────────────────────────────┐
│  좌측 패널 (선수 카드 목록)         │  우측 패널 380px (타순·로테이션·불펜) │
│                                   │  [탭] 타순 / 선발 로테이션 / 불펜    │
│  [필터] 전체 / 투수 / 야수          │                                    │
│                                   │  타순: 1번~9번 슬롯 + 포지션 선택   │
│  ─── 1군 엔트리 (최소20·최대29) ───  │  로테이션: 1선발~5선발 슬롯         │
│  [선수카드] [선수카드] ...          │  불펜: CL·SU·MR 역할 지정           │
│                                   │                                    │
│  ─── 2군 (나머지) ─────────────── │                                    │
│  [선수카드] [선수카드] ...          │                                    │
└───────────────────────────────────┴────────────────────────────────────┘
```

#### 선수 카드 정보

각 선수 카드에서 확인 가능한 정보:
- 선수명 / 포지션 / 외국인 여부
- **OVR** (종합능력치) · **POT** (잠재능력치) 컬러 등급 칩
- 주요 능력치 최대 4개 (코드·수치)

능력치 등급 색상: **S(70+)** 황금 → **A(60+)** 녹색 → **B(50+)** 파란 → **C(40+)** 보라 → **D** 회색

#### 1군 ↔ 2군 이동

- **클릭**: 해당 선수 즉시 토글
- **드래그&드랍**: 선수 카드를 반대 영역으로 드래그

| 항목 | 규칙 |
|------|------|
| 1군 엔트리 | 최소 20명, 최대 29명 선택 |
| 외국인 선수 | 최대 3명 (초과 시 경고) |
| 선발 로테이션 | 1선발~5선발 지정 (투수만) |
| 불펜 역할 | CL(마무리) / SU(셋업) / MR(중간계투) |
| 타순 | 1번~9번 슬롯에 선수·포지션 지정 (드래그&드랍) |

- 완료 시 `GAME_CFG(CFG_KEY='ROSTER_CONFIRMED')` = 1 저장
- 미확정 시 정규시즌 진행 차단

### 3.3 콜업·말소

- 정규시즌 중 언제든 1군·2군 간 이동 가능
- 부상(INJ) 선수는 1군 콜업 불가 — 말소 후 2군 재활 등록
- 콜업 추천 이벤트(EVNT_TYPE_CD = CALL)로 알림 제공

### 3.4 시즌 초기화

- 매 시즌 시작 전 소속 선수 전원을 2군으로 일괄 등록
- 드래프트로 입단한 신인은 지명 즉시 자동으로 2군 등록
- 이후 유저가 로스터 확정 화면에서 1군 엔트리 구성

## 4. 선수 관리

### 4.1 선수 상태

| 코드 | 상태 | 경기 출전 |
|------|------|-----------|
| AT | 활동 | 가능 |
| INJ | 부상 | 불가 |
| RET | 은퇴 | 불가 |
| FA | FA | 불가 (소속팀 없음) |

### 4.2 능력치 조회

- 20~80 스케일 수치와 등급(S+/S/A+/…/D)으로 표시
- 분야별 능력치: 타자(CNT, PWR, RUN, STL, THR), 투수(CTL, BRK, VEL, STM), 포지션 숙련도 포함

### 4.3 숨겨진 능력치 (PLR_HIDE_ABLT)

- 기본적으로 유저에게 숨김
- 스카우터의 JPP/JPA 능력치 수준에 따라 힌트 정확도 결정
- 스카우터 능력이 낮으면 노이즈가 포함된 추정값으로 표시

### 4.4 선수 특성 (PLR_TRT)

경기·성장에 영향을 주는 특성 목록:

| 특성 코드 | 특성명 | 주요 효과 |
|-----------|--------|-----------|
| IRON | 금강불괴 | 부상 확률 대폭 감소 |
| GLAS | 유리몸 | 부상 확률 대폭 증가 |
| RCVR | 회복력 | 부상 회복 기간 단축 |
| LONG | 완투형 | 선발 이닝 연장 |
| ACEM | 에이스 기질 | 포스트시즌·접전 시 능력 상승 |
| CLSR | 마무리 기질 | 세이브 상황 능력 상승 |
| CLTH | 클러치 | 득점권 타점 보정 |
| PWRH | 파워히터 | 홈런 생성 확률 강화 |
| CTMN | 교타자 | 안타 생성 확률 강화 |
| DSPY | 수비 달인 | 수비력 강화 |
| MNTL | 멘탈갑 | 슬럼프 진입 확률 감소 |
| AGED | 노익장 | 노화 시작 연령 +3년 |
| ERLB | 조숙 | 성장 피크 연령 −2년 |
| LATB | 대기만성 | 성장 피크 연령 +3년, 피크 능력치 소폭 상향 |

전체 특성 목록 및 세부 수치 → `Rule.md §2.9`

### 4.5 선수 성장

- 연령대별 성장·유지·노화 단계 존재 (만 18~24 성장, 만 33+ 노화)
- 잠재 능력치(PLR_POT_ABLT): 선수가 도달 가능한 최대 종합 능력치, 변경 불가
- 스프링 캠프 후 성장 이력 → `PLR_GRWTH_LOG` (유저·AI 구단 모두 기록)
- 월간 정산 시 능력치 스냅샷 → `PLR_ABLT_MON` (월별 능력치 히스토리)
- 시즌 종료 시 능력치 스냅샷 → `PLR_ABLT_SSNT` (시즌별 능력치 히스토리)

### 4.6 선수 정보 직접 수정 (단장 권한)

- 선수 상세 화면 이름 옆 스패너(🔧) 아이콘 클릭 시 수정 팝업 표시
- 수정 가능 항목:

| 항목 | 범위 | 설명 |
|------|------|------|
| 피로도(fatg) | 1~100 | 높을수록 경기력 저하 |
| 컨디션(cond) | 1~100 | 낮을수록 경기력 저하 |
| 잠재능력치(potAblt) | 20~80 | 선수의 성장 가능 한계 |
| 개별 능력치(abilities) | 20~80 | 능력치 코드별 수치 |
| 포지션 숙련도(positions) | 20~80 | 포지션별 수비 숙련도 |

- 종합 능력치(PLR_OVRL_ABLT)는 개별 능력치 평균으로 자동 재산출 (직접 수정 불가)
- 수정 완료 시 `PUT /players/{plrId}/player-edit` 호출

## 4-1. 감독·코치 선임 (프리시즌)

### 선임 시점
- 방송국 스폰서 선택 직후 후보 명단이 자동 생성되어 `STFF_CAND` / `STFF_CAND_ABLT` 에 저장된다.
- SeasonPage에서 "감독·코치 선임" 버튼 클릭 시 `StaffHireModal` 표시.

### 선임 규칙

| 구분 | 규칙 |
|------|------|
| 감독(MGR) | 1명 필수 선임 (재계약 또는 신규) |
| 코치(COACH) | 최대 2명 선임 (재계약 또는 신규) |
| 재계약 | 기존 감독·코치가 있으면 재계약 체크박스로 현 인원 유지 가능 |
| 신규 계약 | 감독 후보 6명 / 코치 후보 10명 중 선택 |

- 계약 완료 시 `STFF_TM_CNTRCT` 에 계약 정보 저장, `STFF_CAND` / `STFF_CAND_ABLT` 삭제
- 완료 후 `GAME_CFG(CFG_KEY='STFF_HIRED')` = 1 저장

### 후보 표시 항목
이름, 경력(연수), 종합 능력치(OvrlRtg), 계약금, 연봉, 능력치 목록

## 4-2. 외국인 선수 계약 (프리시즌)

### 계약 기간
- 2월 1일 ~ 2월 10일 (감독·코치 선임 완료 직후 자동 시작)
- 유저가 "그만하기" 버튼을 누르거나 2월 10일이 지나면 종료
- 기간 시작 시 `SSNT_EVNT(EVNT_TYPE_CD='FRGN_OPEN')` 생성 → 버튼으로 계약 팝업 진입

### 후보 명단
- **투수 25명 + 야수 15명 = 총 40명** 전원 표시 (생략 금지)
- 후보는 `FRGN_PLR_CAND` 테이블에 영구 저장 (STATUS_CD: AVAILABLE→SIGNED/AI_SIGNED/REJECTED)
- 능력치는 계약 완료 전까지 숨김 (이름·나이·국적·포지션·과거 스탯만 노출)
- 능력치는 과거 스탯과 상관관계가 있으나 정확한 값은 비공개

### 계약 방식
- 하루 단위로 연봉 오퍼 전송
- 외국인 선수가 다음 날 승낙 또는 거절 (랜덤 + 오퍼 금액 기반 확률)
- 승낙 시 즉시 계약 완료 → `PLR` INSERT(`PLR_FRGN_YN='1'`), `PLR_TM_CNTRCT(FR)` 등록, `PLR_ENTY`(2군) 자동 등록
- **팀당 최대 계약 수**: 서비스 상수 `MAX_FRGN_PLR = 3` (코드 관리)
- 팝업 헤더에 "현재 N명 / 최대 3명" 형식으로 계약 현황 표시

### AI 경쟁
- AI 팀도 동일한 후보 풀에서 외국인 선수에게 매일 자동 오퍼 (날짜 진행 시 `processDailyOffers` 실행)
- 이미 계약된 선수는 다른 팀이 오퍼 불가 — 모달에서 계약 구단명과 함께 `타팀계약(OO)` Chip 표시
- AI 오퍼 금액은 해당 선수의 희망 연봉 기반 자동 산정 (0.85~1.1배)
- 최대 보유(`MAX_FRGN_PLR=3`) 도달 시 모달 상단 안내 Alert + 모든 오퍼 입력 비활성화
- **AI 팀도 팀당 최대 3명** 외국인 선수 보유 제한 (`MAX_FRGN_PLR=3` 동일 적용)

### AI 외국인 선수 평가 및 재계약 (시즌 전)

외국인 후보 풀 생성(`generateCandidatesIfAbsent`) 전에 AI 구단이 기존 외국인 선수의 성적을 평가해 방출 여부를 결정한다.

#### 평가 기준

| 포지션 | 판단 조건 |
|--------|-----------|
| 투수 | 직전 시즌 평균 ERA > 5.50 |
| 야수 | 직전 시즌 평균 타율 < 0.240 |

#### 재계약 여부 결정

1. 기존 외국인 선수의 `PLR_OVRL_ABLT`와 후보 풀 최고 OVRL 비교
2. **후보가 기존 선수보다 8 이상 높으면** → 50% 확률로 방출·재계약 시도
3. 방출 처리: 선수 상태 `REL` 전환, 팀 제외, 라인업/로테이션 자동 정리
4. 방출 후 가장 높은 OVRL 후보에게 자동 오퍼 → 다음 날 승낙 여부 결정

> 성적 기준 미달이더라도 후보 풀에 유의미한 대안이 없으면 방출하지 않는다.

### 외국인 선수 방출 (계약 해지)
- 선수 상세 화면 → 자신 구단의 외국인(`PLR_FRGN_YN='1'`, `PLR_STTS_CD='AT'`) 선수일 때만 "외국인 계약 해지" 버튼 노출
- 확인 다이얼로그 후 `DELETE /api/players/{plrId}/release-foreign` 호출
- 처리 결과: 선수 상태 `REL` 전환, 팀에서 제외, 라인업/로테이션/불펜에서 자동 정리, 동일 시즌 외국인 추가 영입 가능
- 시즌 이벤트(`FRGN` 방출 뉴스) 자동 생성

#### 방출 완료 후 다이얼로그 표시

API 성공 응답(`FrgnPlrReleaseResponse`)을 받으면 다이얼로그 내용이 결과 화면으로 전환:

| 항목 | 내용 |
|------|------|
| 선수명 (plrNm) | 방출된 외국인 선수 이름 |
| 방출일 (releaseDt) | 처리된 날짜 (`CUR_DT`) |
| 해지 선수 연봉 (plrAnslSal) | 방출 선수의 연봉 (만원) |
| 남은 외국인 슬롯 (remainingFrgnSlots) | 방출 후 추가 계약 가능 슬롯 수 |

- 결과 화면에서 "선수 목록으로" 버튼 클릭 → `/players`로 이동

### 외국인 오퍼 버튼 상태 변경
- 오퍼 전송 성공 시 "오퍼 전송" 버튼이 "오퍼 완료" 상태로 비활성화 전환
- 이미 오퍼를 보낸 선수의 행에 "오퍼 완료" 칩/버튼 표시 (재오퍼 불가)
- AI가 먼저 계약한 선수는 "타팀계약(OO)" 칩 표시

## 4-3. 스프링 캠프 (프리시즌)

### 선택 시점
- 감독·코치 선임 완료 후 "스프링 캠프 선택" 버튼 활성화
- 7개 캠프 장소 중 1개 선택

### 캠프 등급 (Tier 1~7)

| Tier | 비용 | 성장 능력치 수 | 능력치당 최대 성장 |
|------|------|----------------|-------------------|
| 1 (하위) | 낮음 | 적음 | 낮음 |
| 7 (상위) | 높음 | 많음 | 높음 |

- 높은 등급일수록 비용 증가, 선수 성장폭 확대
- 비용은 즉시 `TM_FNC_SSNT` 에서 차감

### 스프링 캠프 완료 후
1. `GAME_CFG(CFG_KEY='SPRING_CAMP_DONE')` = 1 저장
2. CUR_DT가 **3월 15일**로 점프
3. 소속 선수 전원 능력치 성장 처리 (`PLR_GRWTH_LOG` 에 성장 기록 저장)
4. `SSNT_EVNT(EVNT_TYPE_CD='GRWTH')` 생성 — **유저 구단의 스프링 캠프 뉴스만** 표시
5. 뉴스 클릭 시 선수별 성장 전/후 능력치 비교 표 제공
6. 로스터 확정 이벤트(EVNT_TYPE_CD=`ROSTER_CONFIRM`) 자동 생성
7. AI 구단도 동시에 캠프 선택 및 성장 처리 (→ §4-3 AI 스프링 캠프 참고)

### AI 스프링 캠프

유저가 캠프를 선택하면 AI 구단들도 자동으로 보유 현금 기준의 Tier를 선택해 스프링 캠프를 진행한다.

| 보유 현금 (CUR_CASH) | 선택 Tier |
|---------------------|-----------|
| 5,000만원 이상 | Tier 7 (최상위) |
| 3,000만원 이상 | Tier 5 |
| 1,000만원 이상 | Tier 3 |
| 500만원 이상 | Tier 2 |
| 500만원 미만 | Tier 1 (최하위) |

- AI 구단도 캠프 비용을 `TM_FNC_SSNT`에서 즉시 차감하고 `TM_FIN_LOG`에 지출 기록
- AI 구단 선수들도 `PLR_GRWTH_LOG`에 성장 이력 기록 (유저 구단과 동일)
- 각 AI 구단의 캠프 결과는 별도 `SSNT_EVNT(EVNT_TYPE_CD='GRWTH')` 뉴스로 생성 (타 구단 현황 확인 가능)

### 7.2 재정 상세 이력 (TM_FIN_LOG)

구단 상세 페이지 → "재정" 탭 → "재정 상세 이력" 서브탭에서 확인.

#### 이력 표 (테이블)

| 컬럼 | 설명 |
|------|------|
| LOG_DT | 발생 일자 |
| LOG_TYPE_CD | `IN`(수입) / `OUT`(지출) |

| 모드 | 차트 유형 | 설명 |

| `expense` | 꺾은선 그래프 (LineChart) | 월별 지출 추이 |
| `stacked` | 누적 영역 그래프 (StackedAreaChart) | 수익 항목별 월별 누적 |
| `sankey` | Sankey 다이어그램 | 재정 흐름 (수입→지출→잔액) |

- recharts `AreaChart` + `stackId` 사용 (stacked 모드)
- recharts `Sankey` 컴포넌트 사용 (sankey 모드, 미지원 시 BarChart 폴백)

## 12. 드래프트

### 12.1 드래프트 방식

| 항목 | 내용 |
|------|------|
| 방식 | 전면 드래프트 (11라운드 × 10팀 = 최대 110명 지명) |
| 지명 순서 | 전년도 최종 순위 역순 (최하위팀이 1순위) |
| 진행 시점 | 오프시즌(OFF) 중 |
| 후보 풀 | 매년 약 400명 자동 생성 |

### 12.2 드래프트 상태 흐름

| 상태 코드 | 상태명 | 설명 |
|-----------|--------|------|
| CREATED | 생성됨 | 드래프트 이벤트 생성 직후 |
| SCOUTING | 스카우팅 | 후보 풀·스카우팅 리포트 생성 완료 |
| READY | 준비완료 | 드래프트 시작 대기 중 |
| IN_PROGRESS | 진행 중 | 지명 진행 중 |
| COMPLETED | 완료 | 전체 라운드 종료 |
| CANCELLED | 취소 | 드래프트 취소 |

### 12.3 스카우팅 리포트

- 각 팀은 후보 선수별로 스카우팅 추정치(현재 능력치·잠재력·예상 라운드·등급·코멘트)를 받음
- 실제 능력치는 비공개 — 스카우팅 시설 등급에 따라 추정치 정확도 결정
- 등급: S+ / S / S- / A+ / A / A- / B+ / B / B- / C+ / C / C- / D

### 12.4 드래프트 보드

- 유저는 후보 선수별로 우선순위(prioOrd), 지명 제외(doNotPick), 메모(memo) 등록 가능
- 드래프트 진행 중 실시간으로 보드 수정 가능

### 12.5 AI 자동 지명

- 유저 픽 차례가 올 때까지 AI가 자동으로 지명
- AI 지명 점수 = 잠재력×0.35 + 현재능력×0.20 + 포지션필요도×0.25 - 부상위험 패널티
- 라운드 초반(1~4라운드)은 잠재력 가중치 증가, 후반은 현재 능력 가중치 증가

### 12.6 유저 픽 및 자동 입단

- 유저가 후보 지명 시 즉시 자동 입단 처리
- 계약 유형: NK(신인 계약), 계약 기간 3년, 연봉은 라운드에 따라 결정
  - 1라운드: 3억 / 2라운드: 1억 / 3~4라운드: 5천만 / 5~7라운드: 3천만 / 8라운드~: 2천만
- 입단 즉시 2군 배치

### 12.7 드래프트 결과 기록

- 매년 드래프트 결과가 DRFT_ORD 테이블에 영구 저장
- 연도별로 어떤 선수가 몇 순위에 어느 팀에 지명되었는지 확인 가능

## 14. 시설 관리

팀 상세 화면 **현황 탭**에서 시설 현황 확인 및 업그레이드를 수행할 수 있다.

### 14.1 시설 유형

| 코드 | 시설명 | 주요 효과 |
|------|--------|-----------|
| TRNG | 훈련 시설 | 선수 능력치 성장률 보정 |
| YUTH | 육성 시설 | 2군 유망주 성장 가속 |
| ANLY | 분석 센터 | 경기 전략 효율, 상대 분석 정확도 |
| SCTG | 스카우팅 센터 | 드래프트 스카우팅 정확도 |
| CAFE | 구단 카페테리아 | 선수 컨디션·피로 회복 보정 |

### 14.2 시설 레벨

- 모든 시설은 **Lv.1 ~ Lv.10** 단계로 관리 (경기장 증축 STDM 제외)
- 게임 시작 시 모든 팀의 모든 시설이 Lv.1로 초기화
- 레벨이 높을수록 해당 시설의 효과가 강화됨

### 14.3 시즌 종료 시 자동 하락

- **매 시즌 종료마다** STDM 제외 시설이 자동으로 1~3단계 랜덤 하락
- 최소 Lv.1 유지 (하락으로 0 이하 불가)
- 유저·AI 팀 모두 동일하게 적용
- 매년 새 기술 도입으로 기존 시설이 상대적으로 노후화되는 규칙
- 상세 로직 → `Rule.md §9-1`

### 14.4 시설 업그레이드

- 업그레이드 버튼 클릭 → 비용·레벨·공사 기간 확인 다이얼로그 표시
- 확인 시 즉시 업그레이드 처리 (시간 대기 없음)
- 업그레이드 비용은 구단 예산에서 차감
- 업그레이드 비용은 `FCLTY_UPGR_COST_CFG` 코드 테이블로 관리 (Lv.1→2 ~ Lv.9→10)
- 이미 Lv.10 최고 레벨인 시설은 업그레이드 버튼 비활성화
- 비용 설정은 개발자 모드에서 관리 가능 (`/dev` → "시설 업그레이드 비용" 탭)

| 항목 | 내용 |
|------|------|
| 비용 관리 | `FCLTY_UPGR_COST_CFG` 테이블 (시설유형 × 레벨별) |
| 이력 저장 | `TM_FCLTY_UPGR` (업그레이드 일시·비용·레벨 변경 기록) |
| API | `POST /api/teams/{tmId}/facility-upgrade` |
