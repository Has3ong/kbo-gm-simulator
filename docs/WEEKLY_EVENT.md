# 주간 처리 (Weekly Event Processing)

> **트리거 조건**: 매주 월요일 (경기 없는 날) `진행하기` 버튼으로 날짜가 월요일로 전환될 때 자동 실행

---

## 처리 순서

### Step 1 — 지난 1주일 성적 요약
| 항목 | 내용 |
|------|------|
| 상태 | 구현 예정 |
| 대상 테이블 | `GAME`, `BATR_GAME_REC`, `PTCH_GAME_REC`, `STND` |
| 처리 내용 | 직전 7일간 팀별 성적(승·패·득실점) 및 주요 선수 성적 집계 |
| 출력 | 주간 리포트 이벤트 생성 (`SSNT_EVNT.EVNT_TYPE_CD = 'NEWS'`) |

---

### Step 2 — 선수 피로도/컨디션 일부 회복
| 항목 | 내용 |
|------|------|
| 상태 | 구현 예정 |
| 대상 테이블 | `PLR_FATG_COND` |
| 처리 내용 | 모든 선수의 피로도를 일정량 감소, 컨디션을 일정량 증가 |
| 규칙 | 주간 회복량: 피로도 -10~-20, 컨디션 +5~+15 (체력(STM) 능력치 반영) |
| 특이사항 | 특성 `GRND(악바리)` 보유 선수는 컨디션 하한 효과 감소 |

---

### Step 3 — 부상 회복 상태 갱신
| 항목 | 내용 |
|------|------|
| 상태 | 구현 예정 |
| 대상 테이블 | `PLR`, `PLR_ENTY` |
| 처리 내용 | `PLR_STTS_CD = 'INJ'` 선수들의 회복 진행도 갱신 |
| 규칙 | 부상 심각도에 따라 회복 기간 차등, 회복 완료 시 `AT`로 전환 |

---

### Step 4 — 로스터 문제 점검
| 항목 | 내용 |
|------|------|
| 상태 | 구현 예정 |
| 대상 테이블 | `PLR_ENTY`, `PLR` |
| 처리 내용 | 1군 엔트리 부족(투수 부족, 포수 없음 등) 자동 감지 |
| 출력 | 문제 발생 시 경고 이벤트 생성 (`SSNT_EVNT.EVNT_TYPE_CD = 'WARN'`) |

---

### Step 5 — 선발 로테이션 다음 주 일정 정리
| 항목 | 내용 |
|------|------|
| 상태 | 구현 예정 |
| 대상 테이블 | `TM_ROTATION`, `GAME` |
| 처리 내용 | 다음 주 경기 일정에 맞춰 선발 투수 로테이션 순서 갱신 |
| 규칙 | 피로도 70 이상 선발투수는 자동 건너뜀 처리 |

---

### Step 6 — 불펜 피로도 경고
| 항목 | 내용 |
|------|------|
| 상태 | 구현 예정 |
| 대상 테이블 | `PLR_FATG_COND`, `TM_BULLPEN` |
| 처리 내용 | 불펜 투수 중 피로도 80 이상 선수 자동 감지 |
| 출력 | 해당 선수 경고 이벤트 생성 (`SSNT_EVNT.EVNT_TYPE_CD = 'WARN'`) |

---

### Step 7 — AI 구단 로스터 자동 조정
| 항목 | 내용 |
|------|------|
| 상태 | 구현 예정 |
| 대상 테이블 | `PLR_ENTY`, `PLR_FATG_COND` |
| 처리 내용 | 유저가 아닌 9개 AI 구단의 부상·피로 선수 자동 엔트리 조정 |
| 규칙 | 피로도 90 이상 선수 자동 2군 강등, 회복 선수 자동 1군 콜업 |

---

### Step 8 — 트레이드 시장 관심도 갱신
| 항목 | 내용 |
|------|------|
| 상태 | 구현 예정 |
| 대상 테이블 | `PLR`, `STND` (성적 기반) |
| 처리 내용 | 성적 부진/과잉 선수에 대한 트레이드 관심도 수치 변동 |
| 출력 | 높은 관심도 선수 트레이드 제안 이벤트 생성 |

---

### Step 9 — 스카우팅 진행도 소폭 갱신
| 항목 | 내용 |
|------|------|
| 상태 | 구현 예정 |
| 대상 테이블 | `DRFT_SCUT_RPT` |
| 처리 내용 | 주마다 스카우팅 보고서 정확도(SCUT_ACCR) 소폭 증가 |
| 규칙 | 스카우팅 시설(SCTG) 등급에 따라 증가량 차등 |

---

### Step 10 — 유저에게 주간 리포트 제공
| 항목 | 내용 |
|------|------|
| 상태 | 구현 예정 |
| 대상 테이블 | `SSNT_EVNT` |
| 처리 내용 | 위 Step 1~9 결과를 종합한 주간 리포트 뉴스 이벤트 생성 |
| 이벤트 유형 | `EVNT_TYPE_CD = 'NEWS'`, `RD_YN = '0'` (미읽음 상태) |

---

## 피로도/컨디션 회복 공식

| 구분 | 회복량 |
|------|--------|
| 피로도 기본 감소 | -15 (주간) |
| STM 능력치 보정 | STM 60 이상: -5 추가, STM 40 이하: +5 (덜 회복) |
| 컨디션 기본 증가 | +10 (주간) |
| 컨디션 랜덤 변동 | ±5 (매일 랜덤) |
| 특성 `WKHS(내구왕)` | 피로도 감소량 1.3배 |
| 특성 `GRND(악바리)` | 컨디션 하락 시 효과 50% 감소 |

---

## 향후 개선 사항

| 우선순위 | 항목 |
|----------|------|
| 높음 | Step 2: 실제 경기 출전 여부 기반 피로도 회복량 차등 적용 |
| 높음 | Step 3: 구체적인 부상 회복 기간 테이블 설계 |
| 중간 | Step 7: AI 구단 트레이드 로직 연동 |
| 낮음 | Step 8: 트레이드 시장 UI 구현 |
