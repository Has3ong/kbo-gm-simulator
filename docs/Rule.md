# KBO 단장 시뮬레이터 — 게임 규칙

## 1. 시즌 진행

### 1.1 시즌 상태 전환

```
PRE (프리시즌)
  → 계약, 스프링캠프, 연습경기, 로스터 구성
  → REG_SSNT_BGNG_DT 도달 시 자동 전환

REG (정규시즌)
  → 하루 루프 반복, 경기 시뮬레이션, 순위 갱신
  → REG_SSNT_END_DT 도달 + 포스트시즌 진출팀 결정 후 전환

POST (포스트시즌)
  → 엔트리 고정, WC→SP→PO→KS 순 진행
  → 한국시리즈 종료 후 전환

CMPL (완료)
  → 기록 확정, 수상 결정, 통계 아카이브

OFF (오프시즌)
  → FA 계약, 방출, 드래프트, 연봉 협상
  → 다음 시즌 PRE 생성
```

### 1.2 하루 진행 루프 (정규시즌)

유저가 진행 버튼을 누르면 서버는 **하루 단위 루프**를 실행한다.

```
1. SSNT.CUR_DT 확인
2. 오늘 경기 목록 조회 (GAME WHERE GAME_DT = CUR_DT)
3. 라인업·로스터 유효성 검사
   - PLR_STTS_CD = INJ·RET 선수 출전 차단
   - 등록 선수 수 규정(28인 엔트리) 확인
4. 경기 시뮬레이션 (§2 참조)
5. 경기 결과 저장 (GAME 업데이트)
6. 선수 기록 저장
   - PLR_BATR_GAME_REC / PLR_PTCH_GAME_REC INSERT
7. 집계 기록 갱신
   - PLR_BATR/PTCH_MON_REC → 해당 월 갱신
   - PLR_BATR/PTCH_SSNT_REC → 시즌 집계 갱신
8. 순위 갱신 (STND 업데이트, §5 참조)
9. 선수 상태 갱신 (§3 참조)
   - 부상 판정, 피로도·컨디션 변화
   - PLR.PLR_STTS_CD 변경 가능
10. 재정 갱신 (TM_FNC_SSNT 업데이트, §6 참조)
    - 입장권 수익, 운영비 등 일별 계산
11. 팬 반응 갱신 (TM_MKT_SSNT 업데이트, §7 참조)
    - PPLT_RTG, FAN_LYLTY 등 경기 결과 반영
12. 이벤트·뉴스 생성 (SSNT_EVNT INSERT)
    - 부상, 기록 달성, 특이사항 등
13. CUR_DT + 1 → 다음 날로 이동
```

### 1.3 멀티-데이 진행 모드

| 버튼 | 동작 |
|------|------|
| 다음 경기까지 | 다음 경기일까지 하루 루프 반복 |
| 일주일 진행 | 7회 하루 루프 반복 |
| 한 달 진행 | 다음 달 1일까지 하루 루프 반복 |
| 정규시즌 끝까지 | REG_SSNT_END_DT까지 하루 루프 반복 |

> 내부적으로는 무조건 **하루 루프 반복** 구조.  
> 부상·이벤트·마감일이 날짜 단위로 빠짐없이 처리된다.

## 2. 경기 시뮬레이션

### 2.1 경기 자동 진행 방식

유저는 경기 과정을 보지 않고 **결과 요약**만 확인한다.  
이닝·타석 단위 처리 없이 팀 전력 점수 → 예상 득점 → 실제 점수 순서로 결과를 생성한다.

```
경기 시작
↓ 양 팀 공격력/선발투수력/불펜전력/수비력 계산 (§2.2)
↓ 예상 득점 계산 (§2.3)
↓ 실제 점수 랜덤 생성 (§2.4)
↓ 승패 결정 / 동점 처리 (§2.5)
↓ 팀 기록 생성 H/HR/BB/SO/SB/E (§2.6)
↓ 선수 기록 분배 (§2.7)
↓ 투수 기록 생성 이닝/승패/세이브 (§2.8)
↓ 경기 종료
```

### 2.2 팀 전력 계산

**팀 공격력** — 라인업 9명 타격 능력치 가중 평균
```
팀 공격력 =
  CNT 평균 × 0.45
  + PWR 평균 × 0.35
  + RUN 평균 × 0.10
  + STL 평균 × 0.05
  + 컨디션 보정 × 0.05
```

**선발투수력** — 선발투수 1명 능력치
```
선발투수력 =
  CTL × 0.30
  + BRK × 0.25
  + VEL × 0.20
  + 보유 구종 수 × 0.15   (P4S/P2S/PCT/PSN/PSL/PCB/PCH/PFK 중 보유 수)
  + STM × 0.10
  - 피로도 보정
```

**불펜 전력** — 상위 5명 평균
```
불펜 전력 =
  상위 불펜 투수 5명 개인투수력 평균
  - 최근 등판 피로도 보정
```

**팀 수비력** — 선발 야수 포지션 적합도 반영
```
팀 수비력 =
  선발 야수 THR 평균 × 0.7
  + 포지션 숙련도(POSN_PRFC_ABLT) 평균 × 0.3
```

### 2.3 예상 득점 계산

```
예상 득점 =
  4.5
  + (우리 공격력 - 50) × 0.08
  - (상대 선발투수력 - 50) × 0.05
  - (상대 불펜 전력 - 50) × 0.03
  - (상대 수비력 - 50) × 0.02
  + 홈팀 보정 (+0.2, 홈팀에만 적용)
  + 컨디션 보정
```

기준값 50 = 능력치 스케일 20~80 기준 KBO 평균 수준.

### 2.4 실제 득점 생성

```
실제 득점 = max(0, round(예상 득점 + 랜덤 변동값))
```

- 랜덤 변동값: −3.0 ~ +4.0 (야구의 높은 변동성 반영)
- 최솟값 0 (음수 불가)

### 2.5 동점 처리 (KBO 룰)

```
정규 9이닝 동점
  ↓
불펜 전력 비교 + 랜덤으로 추가 득점 계산 (최대 12회까지)
  ↓
한쪽 득점 시 → 승패 결정
12회 이후도 동점 시 → 무승부 (T)
```

- 정규시즌에만 무승부 허용
- 포스트시즌은 무조건 승패 결정 (이닝 제한 없음)

### 2.6 팀 기록 생성

점수 확정 후 팀 기록을 생성한다.

| 기록 | 생성 방식 |
|------|-----------|
| 안타 (H) | 득점 + 3 ~ 득점 + 8 범위 내 랜덤 |
| 홈런 (HR) | 팀 PWR 평균 기반 확률 생성 |
| 볼넷 (BB) | 상대 CTL 낮을수록 증가 |
| 삼진 (SO) | 상대 VEL·BRK 높을수록 증가 |
| 도루 (SB) | 팀 STL 평균 기반 확률 생성 |
| 실책 (E) | 팀 수비력 낮을수록 증가 |

> 투수 피안타/피홈런/볼넷/탈삼진 합계는 상대 팀 기록과 정합성 유지 (필수).

### 2.7 선수 기록 분배

팀 총 기록을 선발 9명에게 **가중치 기반**으로 분배한다.

- 라인업 9명 각자에게 타석 3~5, 타수 3~5 배정

| 기록 | 분배 가중치 |
|------|------------|
| 안타 | CNT × 0.7 + PWR × 0.3 |
| 홈런 | PWR × 0.85 + CNT × 0.15 |
| 득점 | CNT × 0.4 + RUN × 0.3 + 타순 보정 × 0.3 (1~3번 가중) |
| 타점 | PWR × 0.45 + CNT × 0.25 + 타순 보정 × 0.30 (3~5번 가중) |

### 2.8 투수 기록 생성

**선발투수 이닝 (IP_OUT 단위, 3 = 1이닝)**
```
선발 이닝 =
  기본 5이닝
  + STM 보정 (+0~2이닝)
  + 선발투수력 보정 (+0~1이닝)
  - 실점 보정 (실점 많을수록 감소)
  - 피로도 보정

범위: 최소 2이닝(6 outs) ~ 최대 9이닝(27 outs)
```

**실점 분배**
```
상대 총 실점 = 선발 실점 + 불펜 실점
선발 실점: 선발 이닝 비율로 배분
자책점 = 실점 (간략화 버전, 비자책 구분 생략)
```

**승리/패전/세이브 판정**

| 판정 | 조건 |
|------|------|
| 선발 승리 | 승리팀 선발이 5이닝 이상 투구 |
| 구원 승리 | 선발 5이닝 미만 → 이후 가장 오래 던진 투수 |
| 선발 패전 | 패배팀 선발 (실점이 가장 많은 투수로 단순 처리) |
| 세이브 | 1~3점 차 승리 + 마지막 등판 투수 |

## 3. 선수 관리

### 3.1 선수 상태 (PLR_STTS_CD)

| 코드 | 상태 | 경기 출전 |
|------|------|-----------|
| AT | 활동 | 가능 |
| INJ | 부상 | 불가 |
| RET | 은퇴 | 불가 |
| FA | FA | 불가 (소속팀 없음) |

### 3.2 부상 시스템

**부상 판정 시점:** 하루 루프 9번 단계 (경기 종료 후)

**기본 부상 확률 계산:**
```
base_prob = 0.003 (경기당 기본 0.3%)
modified_prob = base_prob
  × (1 + fatigue_factor)    # 피로도 가중
  × trait_factor             # 특성 보정 (IRON: ×0.3 / GLAS: ×2.5)
  × position_factor          # 포지션별 위험도
  × age_factor               # 나이별 위험도
```

**부상 등급:**
| 등급 | 회복 기간 | 비고 |
|------|-----------|------|
| 경상 | 3~7일 | 경기 결장 |
| 중상 | 2~6주 | DL 등록 |
| 중증 | 2~6개월 | 시즌 아웃 가능 |
| 시즌 아웃 | 당 시즌 전체 | 수술 등 |

**RCVR(회복력) 특성:** 회복 기간 × 0.6

### 3.3 선수 능력치 성장·노화

**성장 원칙:**
- 만 18~24세: 빠른 성장 구간, 능력치 최대 +3/년
- 만 25~29세: 안정 구간, 미세 성장 또는 유지
- 만 30~32세: 유지·소폭 하락, 특성에 따라 편차 큼
- 만 33세~: 노화 구간, 능력치 최대 −3/년

**특성 보정:**
- AGED(노익장): 노화 시작 나이 + 3년
- ERLB(조숙): 성장 피크 연령 − 2년
- LATB(대기만성): 성장 피크 연령 + 3년, 피크 능력치 소폭 상향

**스냅샷 저장:**
- 매 시즌 시작: PLR_ABLT_SSNT에 현재 능력치 스냅샷
- 매월 말: PLR_ABLT_MON에 월간 능력치 스냅샷

**잠재 능력치 (PLR_POT_ABLT):**
- 선수가 도달할 수 있는 최대 종합 능력치
- 변경 불가 (innate)
- 현재 종합 능력치(PLR_OVRL_ABLT)가 잠재 능력치를 초과할 수 없음

### 3.4 숨겨진 능력치 (PLR_HIDE_ABLT)

- 유저에게는 해당 능력치가 높은지/낮은지 힌트만 표시
- 스카우터 능력치(JPP: 성장가능성 판단, JPA: 현재능력 판단)에 따라 정확도 결정
- 스카우터 능력치가 낮으면 표시값에 노이즈 추가

| 코드 | 게임 내 영향 |
|------|--------------|
| FCN (집중력) | 슬럼프 빈도, 중요 상황 집중 보정 |
| DRV (승부욕) | 접전 상황 능력치 미세 보정 |
| LDR (리더십) | 주변 선수 사기 영향 |
| IRK (성마름) | 심판 판정 불만 시 퇴장 확률 증가 |
| CST (일관성) | 경기별 능력치 분산 축소 |
| DRT (더티 플레이어) | 빈볼·위험한 플레이 빈도 |
| BGM (큰 경기 남자) | 포스트시즌·위기 상황 능력치 보정 |
| AMB (야망) | 능력치 성장률 보정 |
| PRF (완벽주의) | 훈련 효과 보정, 슬럼프 회복 속도 |
| SPT (스포츠맨십) | 팬 이미지, 구단주 호감도 |
| PAT (인내) | 슬럼프 기간 단축 |

| 4 | 국외 우수 | 4 | 2 | 3,500 |
| 5 | 미국 플로리다 | 5 | 3 | 5,000 |
| 6 | 일본 오키나와 | 5 | 4 | 6,500 |
| 7 | 도미니카 | 6 | 4 | 8,000 |

- `growthAbltCount`: 성장 가능한 능력치 수 (랜덤 선택)
- `maxGrowthPerAblt`: 능력치당 최대 성장치

### 성장 계산 로직
```
선수별 처리:
  캠프 능력치 수만큼 능력치 코드를 랜덤 선택
  각 능력치에 대해:
    성장 확률:
      현재값 ≤ 40  → 100%
      현재값 ≤ 65  → 선형 감소 (65기준 50%)
      현재값 > 65  → 급격 감소 (75: 20%, 80: 0%)
    성장량 = Random(1 ~ maxGrowthPerAblt) — 확률 통과 시만
    신규값 = min(현재값 + 성장량, PLR_POT_ABLT 상한)
    PLR_ABLT.ABLT_VAL 갱신
    성장 내역 → PLR_GRWTH_LOG 저장 (유저 팀 · AI 팀 모두)
  종합능력치(PLR_OVRL_ABLT) 성장 동일 로직으로 별도 계산
```

### 성장 기록 (PLR_GRWTH_LOG)
| 컬럼 | 설명 |
|------|------|
| PLR_ID | 선수 ID |
| SSNT_YR | 시즌 연도 |

| ABLT_VAL_BFR | 성장 전 수치 |

> 유저 팀·AI 팀 구분 없이 모든 성장 내역을 `PLR_GRWTH_LOG`에 기록한다.  
> 선수 상세 > 성장 이력 탭에서 조회 가능하다.

## 5. 순위 계산 (STND)

### 5.1 승률·게임차 공식

```
PCT = W / (W + L)          -- 무승부 제외 기준
GB  = ((1위팀 W - 해당팀 W) + (해당팀 L - 1위팀 L)) / 2
```

- 무승부(T)는 승·패 어느 쪽도 아닌 별도 집계
- 1위 팀의 GB = 0.0
- W + L = 0 이면 PCT = 0.0000

### 5.2 순위 정렬 기준

1. PCT 높은 순
2. W 많은 순
3. RUN_DIFF(득실차) 높은 순
4. RS(득점) 많은 순
5. TM_ID 낮은 순 (동률 최종 처리)

### 5.3 연속 기록 갱신 (STRK_TYPE / STRK_CNT)

```
경기 결과 = 승
  → STRK_TYPE = 'W', STRK_CNT +1 (기존 W 연속이면) 또는 1로 리셋

경기 결과 = 패
  → STRK_TYPE = 'L', STRK_CNT = 1 (기존 W이면 리셋)

경기 결과 = 무
  → STRK_TYPE = 'D', STRK_CNT = 1 (무승부는 연속 카운트 초기화)
```

### 5.4 최근 10경기 갱신 (L10_W / L10_L / L10_T)

- TM_GAME_REC에서 해당 팀의 최근 10경기 RSLT(W/L/D) 개수 집계
- 경기가 10경기 미만이면 실제 경기 수 기준

### 5.5 포스트시즌 진출 상태 (PSTSSNT_STTS)

| 값 | 전환 조건 |
|----|-----------|
| UNDC | 기본값 (시즌 시작) |
| ELIM | 수학적으로 포스트시즌 진출 불가 확정 |
| CLWC | 4~5위 와일드카드 진출 확정 |
| CLPS | 포스트시즌 진출 확정 (순위 유동) |
| CL1P | 정규시즌 1위 확정 |
| CHMP | 한국시리즈 우승 |

> MVP 초기 버전: 정규시즌 종료 후 최종 순위 기준으로 일괄 갱신. 시즌 중 실시간 확정은 추후 추가.

### 5.6 팀 분위기·만족도 갱신 (TM_MORL / FAN_STSFCTN / OWN_STSFCTN)

| 수치 | 변화 조건 |

| TM_MORL | 연승 시 +1~3 / 연패 시 −1~3 / 범위 20~80 |

## 7-1. 외국인 선수 계약 규칙

### 계약 가능 기간
- **시작**: 감독·코치 선임 완료 직후 (날짜가 **2월 1일**로 자동 이동)
  - 유저 선임 완료 시 AI 팀들도 동시에 감독·코치 자동 선임
  - `FRGN_OPEN` 뉴스 이벤트가 생성되며 계약 기간을 안내
- **종료**: 2월 10일 자동 종료 또는 유저가 "그만하기" 버튼 클릭 시

### 진행 흐름
```
방송국 스폰서 선택
  → 유저 감독·코치 후보 생성 + STFF 뉴스("감독·코치 선임 필요")
유저 감독·코치 선임 완료
  → AI 팀 감독·코치 자동 선임 (동시 처리)
  → STFF 뉴스: 유저팀 + 모든 AI팀 선임 현황 통합 표시
  → 외국인 선수 후보 40명 사전 생성
  → 날짜가 2월 1일로 자동 이동
  → FRGN_OPEN 뉴스: 계약 기간 안내 + "외국인 용병 계약하기" 버튼
2월 1일 ~ 2월 10일: 매일 오퍼 처리
  → 유저/AI 오퍼 제출
  → 다음 날 진행하기 시 결과 반영 (승낙/거절)
2월 10일 이후: 계약 기간 종료, 오퍼 불가
```

### 후보 풀 생성
- 투수 25명 + 야수 15명 = **40명 전체 반드시 표시** (생략 불가)
- 능력치는 계약 전 숨김 — 이름·나이·국적·포지션·과거 스탯만 공개
- 계약 체결 후 능력치 공개 및 `PLR_ABLT` 에 저장

### 연봉 오퍼 및 승낙 규칙
```
유저가 특정 선수에게 연봉 오퍼 전송
  → 다음 날 진행하기 시 해당 선수 응답 처리
     승낙 확률:
       오퍼 >= 희망 연봉 × 1.0 → 95%
       오퍼 >= 희망 연봉 × 0.9 → 75%
       오퍼 >= 희망 연봉 × 0.8 → 50%
       오퍼 >= 희망 연봉 × 0.7 → 25%
       그 미만               → 5%
     승낙 시 → PLR, PLR_TM_CNTRCT, PLR_ENTY(2군), FRGN 뉴스 생성
     거절 시 → FRGN 뉴스(거절 알림) 생성, 다음날 재오퍼 가능
```

- AI 팀도 동일한 풀에서 매일 1~2개 오퍼 → 먼저 승낙된 선수는 타팀 오퍼 불가
- 팀당 최대 3명 계약, 3명 초과 오퍼 불가 (AI 팀도 동일 제한)
- 이벤트 타입: `FRGN_OPEN`(기간 시작), `FRGN`(계약 체결/거절/방출), `FRGN_OVER`(초과 경고)

### AI 팀 외국인 선수 평가 및 재계약

외국인 선수 후보 풀 생성 직전(매 시즌 1/15 시점)에 모든 AI 팀이 보유 외국인을 평가한다.

#### 평가 기준

| 타입 | 부진 판정 조건 |
|------|---------------|
| 투수 | 직전 시즌(ssntYr-1) `PLR_PTCH_SSNT_REC.ERA` > 5.5 |
| 타자 | 직전 시즌(ssntYr-1) `PLR_BATR_SSNT_REC.AVG` < 0.240 |

> 직전 시즌 기록이 없으면(신규 계약 등) 평가를 생략하고 유지한다.

#### 방출 결정 로직
```
성적 부진 판정 시:
  현재 선수의 PLR_OVRL_ABLT 확인
  같은 타입(P/B) AVAIL 후보들의 평균 능력치 기반 최고 OVRL 계산
  후보 최고 OVRL > 현재 선수 OVRL + 8 이면 → 방출 후보
  방출 후보 확정 시 50% 확률로 실제 방출 실행
```

#### 방출 처리
- PLR 상태: `PLR_STTS_CD = 'REL'`, `TM_ID = NULL`
- PLR_TM / PLR_TM_CNTRCT 종료 처리
- PLR_ENTY / TM_LINEUP / TM_ROTATION / TM_BULLPEN 삭제

- `SSNT_EVNT(EVNT_TYPE_CD = 'FRGN')` 뉴스 생성 — "성적 부진으로 방출" 메시지

#### 재계약
- 방출 후 AI 팀은 기존 `makeAiOffers` 로직으로 후보 풀에서 새 외국인을 영입할 수 있다.
- 영입 시점: 2/1 ~ 2/10 오퍼 기간 중 자동 처리

### AI 팀 감독·코치 자동 선임
- 유저 선임 완료와 동시에 모든 AI 팀 자동 처리
- 각 팀 감독 1명, 코치 2명 랜덤 생성 (경력 1~20년, 능력치 5~20)

### 시설 종류 및 최대 레벨

| 시설 코드 | 시설명 | 최대 레벨 | 주요 효과 |

> 경기장 증축(STDM)은 레벨 개념이 없으며 별도 `STDM_EXPN` 테이블로 관리.

### 시설 업그레이드 재정 게이트

- 업그레이드 비용은 만원 단위 (`FCLTY_UPGR_COST_CFG.UPGR_COST`).
- 업그레이드 시 `TM_FNC_SSNT.CUR_CASH >= UPGR_COST` 인 경우에만 실행되며,
  성공 시 같은 트랜잭션에서 `CUR_CASH -= UPGR_COST`, `CUR_FCLTY_COST += UPGR_COST`
  로 원자적 차감된다.
- 잔액 부족 시 `IllegalStateException`을 던져 업그레이드를 차단한다.
- 프론트 구단 상세 화면(`TeamDetailPage`)에서도 `CUR_CASH < UPGR_COST` 일 때
  업그레이드 버튼이 비활성화되고 부족 금액을 툴팁으로 안내한다.

### GRSS(경기장 잔디) 효과

- 매 경기 종료 시 `GameResultService.applyTeamFatigue`에서 적용.
- 잔디 레벨 `L` (1~5) 에 대해 `step = L − 1`:
  - `fatgDelta = max(1, baseFatg − round(step × 0.5))`
    - 투수 기본 12, 야수 기본 5 → 최고 레벨 시 투수 10, 야수 3
  - `condDelta = (baseRandom(-3 ~ +4)) + round(step × 0.4)`
    - 최고 레벨 시 변동 범위가 -1 ~ +6 으로 상향
- 효과는 의도적으로 작게(레벨당 ~5% 미만) 설계되어, 누적 시즌이 지나야
  체감 가능한 수준. "아주 조금" 완화한다는 게임 설계 의도와 일치.
- 부상 위험 감소는 피로도가 누적되어 발생하는 후속 부상 이벤트의 빈도를
  자연스럽게 낮추는 형태로 작동한다 (별도 in-game 부상 발생 로직이 추후
  추가될 경우 `grassFactor`를 동일 식으로 곱한다).

### 시즌 종료 시 시설 자동 하락

**발생 시점**: 매 시즌 종료 처리(`SeasonEndService`) 시 자동 실행.

**규칙**:
- STDM(경기장 증축) 제외한 모든 시설에 적용
- 각 시설마다 독립적으로 랜덤 1~3단계 하락
- 최소 1단계 유지 (`GREATEST(1, FCLTY_LVL - drop)`)
- 유저 팀·AI 팀 구분 없이 전 구단 동시 적용

**배경**: 매년 새로운 기술이 등장함에 따라 기존 시설이 상대적으로 노후화되는 게임 내 규칙.

**SQL 로직**:
```sql
-- 각 행(시설)마다 개별 랜덤값 적용
UPDATE TM_FCLTY
SET FCLTY_LVL = GREATEST(1, FCLTY_LVL - ?)   -- ? = 1~3 랜덤
WHERE TM_ID = ? AND FCLTY_TYPE_CD != 'STDM'
```

### 시설 업그레이드 비용 관리

- `FCLTY_UPGR_COST_CFG` 테이블: 시설 종류별 × 레벨별(1→2, 2→3, …, 9→10) 비용 코드 관리
- 개발자 모드(`/dev` 페이지 → "시설 업그레이드 비용" 탭)에서 인라인 편집 후 일괄 저장 가능
- API: `GET/PUT /api/dev/facility-costs`

## 12. 기록 누적 처리

### 12.1 처리 흐름

경기 시뮬레이션 완료 후 아래 순서로 처리한다.

```
경기 결과 저장 (GAME 업데이트)
↓
TM_GAME_REC 저장 (팀 경기 기록)
↓
PLR_BATR_GAME_REC / PLR_PTCH_GAME_REC 저장 (선수 경기 기록)
↓
TM_SSNT_REC 누적 (팀 시즌 기록)
↓
TM_MON_REC 누적 (팀 월간 기록)
↓
PLR_BATR_SSNT_REC / PLR_BATR_TM_SSNT_REC 누적
↓
PLR_BATR_MON_REC 누적
↓
PLR_PTCH_SSNT_REC / PLR_PTCH_TM_SSNT_REC 누적
↓
PLR_PTCH_MON_REC 누적
↓
STND 갱신 (순위·상태)
↓
GAME.IS_AGGRG_YN = 'Y', AGGRG_DT = 현재 시각
```

### 12.2 누적 원칙

- **경기 기록이 원본**: PLR_BATR/PTCH_GAME_REC, TM_GAME_REC가 source of truth
- **월간·시즌 기록은 집계**: 원본으로부터 더한 집계값. 오류 시 원본 기준으로 재집계 가능
- **파생 지표는 저장 안 함**: BA, OPS, ERA, WHIP 등은 저장하지 않고 조회 시 계산

### 12.3 중복 누적 방지

경기 누적 전 반드시 `GAME.IS_AGGRG_YN = 'N'` 확인.  
처리 완료 후 `IS_AGGRG_YN = 'Y'`, `AGGRG_DT = NOW()` 로 갱신.

### 12.4 트랜잭션 처리

한 경기 누적은 하나의 트랜잭션으로 묶는다.  
중간 실패 시 전체 롤백.

```
BEGIN
  1. IS_AGGRG_YN = 'N' 확인 (이미 'Y'면 중단)
  2. TM_GAME_REC 저장
  3. 선수 경기 기록 저장
  4. 팀 시즌·월간 기록 누적
  5. 선수 시즌·월간 기록 누적
  6. STND 갱신 (순위·상태·연속·L10·분위기)
  7. IS_AGGRG_YN = 'Y', AGGRG_DT = NOW()
COMMIT
```

### 12.5 트레이드 선수 기록 분리

시즌 중 팀을 옮긴 선수는 팀별로 기록을 분리 저장한다.

- `PLR_BATR_TM_SSNT_REC` / `PLR_PTCH_TM_SSNT_REC`: TM_ID별 시즌 기록
- 선수의 시즌 전체 기록은 조회 시 팀별 기록을 합산

### 12.6 재집계 기능

기록 오류 복구를 위해 재집계 기능을 반드시 구현한다.

```
1. 해당 시즌 집계 테이블 초기화
   (TM_SSNT_REC / TM_MON_REC / STND 일부 / PLR_*_SSNT_REC / PLR_*_MON_REC)
2. 해당 시즌 모든 GAME 경기 중 IS_AGGRG_YN='Y' 경기 날짜 순 재처리
3. STND 전체 재계산
4. IS_AGGRG_YN 재설정
```

원본 테이블(GAME, TM_GAME_REC, PLR_BATR_GAME_REC, PLR_PTCH_GAME_REC)은 삭제하지 않는다.

## 13. 파생 지표 계산 공식

파생 지표는 DB에 저장하지 않고 조회 시 계산한다.

### 13.1 타자 파생 지표

```
BA  = H / AB
OBP = (H + BB + HBP) / (AB + BB + HBP + SF)
SLG = 총루타 / AB
    총루타 = (H - DOBL - TRPL - HR)      -- 단타
            + DOBL × 2
            + TRPL × 3
            + HR × 4
OPS = OBP + SLG
```

### 13.2 투수 파생 지표

```
ERA  = ER × 27 / IP_OUT          (IP_OUT: 아웃카운트 단위)
WHIP = (BB + H) × 3 / IP_OUT
K/9  = SO × 27 / IP_OUT
BB/9 = BB × 27 / IP_OUT
```

### 13.3 규정 타석·이닝 기준

| 구분 | 계산식 | 예 (100경기 기준) |
|------|--------|------------------|
| 규정 타석 | 팀 경기 수 × 3.1 | 310 타석 |
| 규정 이닝 | 팀 경기 수 × 1.0이닝 = × 3 (IP_OUT) | 300 아웃카운트 |

## 15. 드래프트 AI 지명 로직

각 AI 팀은 자기 차례에서 풀(`DRFT_PLR`) 중 `IS_PICK_YN='N' AND IS_EXCL_YN='N'` 인 선수에 대해
**팀별 스카우팅 리포트(`DRFT_SCUT_RPT`) 기준의 보기 능력치**와 **현재 로스터의 포지션 부족도**를
결합한 점수를 산출하고, 최고점 선수를 지명한다.

> AI는 실제 능력치(`ACT_OVRL_ABLT` / `ACT_POT_ABLT`)를 보지 않고,
> 해당 팀의 스카우팅 추정값(`EST_OVRL_ABLT` / `EST_POT_ABLT`)을 기준으로 평가한다.
> 즉, 같은 선수라도 팀마다 다른 점수가 나올 수 있다.

### 15.1 평가 점수 공식

```
score = abilityScore + posnNeed × 0.6 − riskPenalty + variance
```

| 항목 | 계산 |
|------|------|
| `abilityScore` | `estPot × potWeight + estOvrl × ovrlWeight` |
| `estPot`       | 해당 팀이 본 잠재 능력치 (`EST_POT_ABLT`, 없으면 `ACT_POT_ABLT`) |
| `estOvrl`      | 해당 팀이 본 현재 능력치 (`EST_OVRL_ABLT`, 없으면 `ACT_OVRL_ABLT`) |
| `posnNeed`     | 팀의 해당 포지션 부족도 (15.3 참조) |
| `riskPenalty`  | `INJ_RSK / 4` (1~20 → 0.25~5점) |
| `variance`     | `ThreadLocalRandom` 으로 -2 ~ +2 사이 균등 분포 |

### 15.2 라운드별 가중치

초반 라운드에서는 잠재력, 후반 라운드에서는 즉전감을 중시한다.

| 라운드 | `potWeight` | `ovrlWeight` |
|--------|-------------|--------------|
| 1~3R   | 0.55        | 0.25         |
| 4~7R   | 0.40        | 0.40         |
| 8R~    | 0.25        | 0.55         |

### 15.3 포지션 부족도(`posnNeed`)

대표 포지션 그룹(`REPR_POSN_CD`) 단위로 계산한다.

| 코드 | 포지션 | 목표 인원 |
|------|--------|-----------|
| 10   | 투수   | 12명       |
| 20   | 포수   | 3명        |
| 21   | 내야수 | 6명        |
| 22   | 외야수 | 5명        |

```
abltGap = max(0, 55 − 해당_포지션_평균_OVRL)
cntGap  = max(0, 목표_인원 − 현재_인원) × 2.5
posnNeed = abltGap + cntGap
```

- `해당_포지션_평균_OVRL`: 팀 활성 선수(`PLR.PLR_STTS_CD='AT'`)의 `REPR_POSN_CD`별 평균 `PLR_OVRL_ABLT`
  - `PLR_TM_CNTRCT` 진행 중 계약(`FA_CNTRCT_END_DT >= CURDATE() OR NULL`)을 기준으로 `REPR_POSN_CD` 매핑
  - 해당 포지션 선수가 0명이면 평균을 0으로 간주 → `abltGap` = 55
- `현재_인원`: 동일 기준의 활성 선수 수
- `posnNeed` 값은 보통 0~30 범위. `posnNeed × 0.6` 보정으로 능력치 차이를 압도하지 않도록 한다.
- 같은 드래프트 안에서 이미 지명된 선수는 `executePick`이 `PLR` 테이블에 즉시 입단 처리하므로
  자동으로 다음 픽의 부족도에 반영된다.

### 15.4 설계 의도

- **좋은 선수 우선** — 점수의 주된 동인은 `abilityScore`(범위 ~25~70). 부족 포지션 보정은
  유사 능력치 선수 간 타이브레이커 역할로 동작한다.
- **포지션 균형** — 동일 능력치 후보가 있을 때 부족 포지션 선수에게 +9점(`posnNeed=15 × 0.6`)
  까지 가산되어, 깊이가 부족한 포지션부터 채워가는 경향을 만든다.
- **약한 변동성(±2점)** — 결정론을 피하면서도 매 픽이 거의 동일한 우선순위로 흐르도록 한다.
- **스카우팅 정확도 반영** — `DRFT_SCUT_RPT.ACCRCY`로 `EST_OVRL_ABLT`·`EST_POT_ABLT` 자체에
  팀별 노이즈가 들어가 있으므로, AI의 평가는 자연스럽게 팀의 스카우팅 역량에 의존한다.
